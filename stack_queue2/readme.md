## 栈和队列的特性
 看成操作受限的数组
 栈只能在栈顶进行操作，数组的尾部操作。LIFO 后进先出
 队列只能在队尾入队，在队头出队。FIFO 先进先出
 也可以用链表来表达

## 链表
 - 数组增加元素，arr.length >= capacity 时，需要扩容
   数组只有扩容和初始化的时候要开销，其他时候没有
 - 链表 每次都申请 开销平均
 数据的规模
 小 数组
 大 链表
 数组的优势 遍历，按索引取元素，缺点是增删，移动位置和申请空间
 链表优势 增删 (前驱节点，不要坐过站) 缺点是找到相应节点

 链表和数组相似 都是**有序**的列表(list) 都是线性结构(有且仅有一个前驱节点，有且仅有一个后继节点)
 array 索引
 linked list head -> node ...... -> tail

 链表中，数据单位节点，指针
 节点和节点的分布是不连续的，是离散的
 js 用对象字面量
 {
  val: 1,
  next: {
    val: 2,
    next: null
  }
 }

 head ? 节点
 访问链表任何一个元素，都是从起始节点开始，逐个访问next，直到目标节点为止。tail 节点的next 指向 null
 
## 链表元素的添加
 本质是对next指针操作
 1. 找到目标节点的前驱节点
 2. 改变前驱节点的next 指向，指向新节点
 3. 新节点的next 指向目标节点
 
## 删除
 1. 找到目标节点的前驱节点
 2. 改变前驱节点的next 指向，指向目标节点的后继节点
 3. 目标节点的next 指向 null
 

 假设数组的长度是n，因删除/增加操作导致需要移动的元素数量，
 随着数组长度n 的增大而增大，呈现一个线性关系。
 因此，增加，删除的操作都是线性时间复杂度 O(n)

 - js数组特性
 JS 数组未必是真正的数组，如果每一项类型一致，纯数组，连续内存
 如果每一项类型不一致，不是纯数组，一段非连续内存，此时JS 数组不在具有数组特征，
 底层是用hashTable 分配内存空间。

 链表有高效的增删操作
 O(n) 固定的
 明确要插入或删除的目标位置，接着只需要改变目标节点及其前驱、后继节点的指针指向。(O(1))
 麻烦的访问操作 (O(n))
